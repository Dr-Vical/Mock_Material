# Response

## figma2xaml 정리

**Figma 플러그인**으로, Figma 디자인을 **WPF ResourceDictionary XAML**로 변환합니다.

### 코드를 뽑으려면 이런 상태여야 함

| 조건 | 설명 |
|------|------|
| **레이어 1개만 선택** | Frame / Group / Component / Instance 중 하나 |
| **다중 선택 불가** | 반드시 단일 선택 |
| **벡터 기반 디자인** | 도형, 경로, 원, 사각형, 선 등 벡터 요소만 변환됨 |

### 변환 결과물

| Figma | → XAML |
|-------|--------|
| 도형 | `GeometryDrawing` |
| 패스 | `Path` |
| 원/타원 | `EllipseGeometry` |
| 사각형 | `RectangleGeometry` |
| 선 | `LineGeometry` |
| 회전 | `RotateTransform` |
| 그라데이션 | `LinearGradientBrush` / `RadialGradientBrush` |
| 윤곽선 | `Pen` |

### 사용 흐름
```
Figma에서 Frame/Component 1개 선택
  → 플러그인 실행 → "Get code" 클릭
  → XAML 코드 생성 → "Copy"
  → Visual Studio에서 ResourceDictionary로 붙여넣기
```

### 한계

- **아이콘/벡터 그래픽 전용** — `GeometryDrawing`, `Path` 중심 출력
- **레이아웃/컨트롤 변환 안 됨** — `Grid`, `StackPanel`, `Button`, `TextBox` 같은 WPF 컨트롤로는 변환하지 않음
- **텍스트, DataGrid, Ribbon 등 미지원**
- 마지막 업데이트가 **2021년**으로 유지보수 중단 상태

### 결론

이 플러그인은 **Figma 아이콘/벡터 에셋 → XAML DrawingImage 리소스** 추출용입니다. 현재 프로젝트처럼 DataGrid, Ribbon, AvalonDock 기반의 **실제 UI 레이아웃 변환은 불가능**합니다.

Figma 디자인을 WPF 코드로 변환하려면 현재처럼 **Figma MCP(`get_design_context`)로 스크린샷 + 구조를 읽고 → 직접 XAML로 작성**하는 방식이 가장 현실적입니다.
