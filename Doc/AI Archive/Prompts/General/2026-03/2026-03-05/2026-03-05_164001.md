# Response

코드를 분석한 결과, **2가지 근본적인 문제**가 있습니다.

## 문제 1: A/B/C/D 전부 같은 데이터를 공유

`MainWindow.xaml.cs:152` — 모든 패널이 같은 ViewModel을 DataContext로 받습니다:

```csharp
DataContext = vm,  // 4개 패널 모두 동일한 vm
```

`CompareParameterPanel.xaml:90` — DataGrid가 바인딩하는 소스:

```xml
ItemsSource="{Binding Parameters}"  ← vm.Parameters 하나를 공유
```

**즉, A/B/C/D가 각기 다른 아이템을 보여줄 수 없는 구조**입니다. 현재 4분할 해도 전부 같은 리스트입니다.

## 문제 2: 로딩이 느린 원인

`MainWindowViewModel.cs:139-167` — `LoadParametersForNode`:

```csharp
Parameters.Clear();  // CollectionChanged 1회

foreach (var p in csvParams)
{
    if (FavoriteParameters.Any(f => f.FtNumber == p.FtNumber))  // O(n×m) 선형탐색
        p.IsFavorite = true;

    p.PropertyChanged += OnParameterPropertyChanged;
    Parameters.Add(p);  // ← CollectionChanged가 매 건마다 발생!
}
```

느린 이유 3가지:

| 원인 | 설명 |
|------|------|
| **개별 Add()** | 파라미터 수백 개를 하나씩 `Add()` → DataGrid가 **매번 레이아웃 재계산** |
| **O(n×m) Favorites 탐색** | 파라미터마다 `FavoriteParameters.Any()` 선형 탐색 |
| **4패널 동시 갱신** | 패널 4개가 같은 `Parameters`를 바인딩 → Add 1회에 DataGrid 4개가 동시 반응 |

---

이 두 문제를 함께 고칠까요? 방향은:

1. **패널별 독립 `ObservableCollection<Parameter>`** — 각 패널이 자체 파라미터 리스트를 가짐 (다른 트리노드 선택 가능)
2. **일괄 로딩** — `Parameters.Add()` 반복 대신 리스트를 한번에 교체 (`= new ObservableCollection<>(list)` 또는 `AddRange`)
3. **HashSet으로 Favorite 조회** — O(1) 검색
