## 변경 요약

### 구조 변경

| 기존 | 변경 후 |
|------|---------|
| 4패널 → 1개 `vm.Parameters` 공유 | 각 패널 → 자체 `PanelParameters` (독립 컬렉션) |
| Add() 반복 → DataGrid 4개 동시 갱신 | `new ObservableCollection<>(list)` 일괄 할당 |
| 4패널 모두 같은 값 | A=기준값, B/C/D=변동폭 증가 |

### Mock 데이터 생성 규칙 (`CsvParameterLoader.LoadForPanel`)

| 패널 | 변동폭 | 설명 |
|------|--------|------|
| **A** | 0% | Default 그대로 (기준 장비) |
| **B** | ±15% | 약간의 드리프트 (가벼운 튜닝 차이) |
| **C** | ±35% | 중간 수준 변동 (다른 모터 특성) |
| **D** | ±60% | 큰 변동 (테스트/경계 데이터) |

- Min/Max 범위 내에서 Clamp
- Default의 소수점 자릿수 보존 (정수면 정수, 소수면 동일 자릿수)
- 패널 ID 기반 시드로 결정적(deterministic) 랜덤 → 같은 패널은 항상 같은 값

### 성능 개선

- 이전: `Parameters.Add()` × 수백 건 → DataGrid 4개가 매번 레이아웃 재계산
- 이후: `new ObservableCollection<>(list)` 한번에 할당 → UI 1회만 갱신
